Taskwarrior Statistics

Interesting statistics from the taskwarrior web site and repository. These data form a baseline against which to measure changes. This page will be updated after the end of each month.

Visits to taskwarrior.org, tasktools.org and assorted web sites.
Note that only downloads directly from taskwarrior.org are now recorded. SourceForge is no longer a contributor. Note that Taskserver and the PDFs now account for half the download traffic.
Total downloads by version. This chart is indirectly measuring the time between releases, and shows that 1.9.4 and 2.3.0 were the primary version for a year. More recently multiple versions are simultaneously available. HomeBrew, for example, downloads the tarballs from us, and the version obtained is dependent on the freshness of the local brew repository.

Development began in 2006, but used CVS, then Subversion. That history was not brought into git, but consisted of 790 or so commits. The largest spike represents 2.4.5, which has been undergoing a test suite migration, and parser improvements.
It is expected that there will ultimately be 10,000 unit tests. The more unit tests we add, the harder it becomes to break task and not notice. More tests should result in fewer bugs, except in the areas of new features, which tend to be buggy at first, until the bugs are found and the unit tests catch up. The recent drop represents the test suite migration to Python, which included the removal of many duplicate and overlapping tests.
The add rate and resolve rate usually track each other well, except in the lead up to a release, where the testing is greater. There is always a shift toward fixing right before a release.
This is the distribution of operating systems for our site visitors.
This char shows the downloads by version, and indicates the rate of adoption. Downloading old versions is caused by out-of-date links around the web, and when a release occurs mid-month. Interestingly, the 2.3.0 download rate is increasing, despite five subsequent releases, and is likely caused by HomeBrew.
The test suite consists of test script/binaries written in various languages. As of August 2015, there are no Perl tests remaining. There are now a handful of Bash tests, which will grow, but will not become a significant part of the test suite. Instead it is for quickly capturing tests, which eventually migrate into feature-specific tests.
Social membership numbers are still rising. G+ is dead.
