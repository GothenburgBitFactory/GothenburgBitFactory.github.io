Taskwarrior Statistics

Interesting statistics from the taskwarrior web site and repository. These data form a baseline against which to measure changes. This page will be updated after the end of each month.

Visits to taskwarrior.org, tasktools.org and assorted web sites. With the rollout of the Atlassian tools on multiple Rackspace machines, we expect to see greater traffic to the *.tasktools.org domain, and lesser traffic to takswarrior.org.
Note that only downloads directly from taskwarrior.org and sourceforge.com are recorded. Note that Vit, Taskserver and the PDFs now account for half the download traffic.
Total downloads by version. 1.9.4 was available for a whole year while 2.0 was developed, which explains the high download count for a single version.
Total downloads by minor version.
Total downloads by major version.
Development began in 2006, but used Subversion. That history was not brought into git, but consisted of 790 or so commits. The largeest spike represents 1.8.0, which was almost a complete rewrite.
It is expected that there will ultimately be 15,000 unit tests. The more unit tests we add, the harder it becomes to break task and not notice. More tests should result in fewer bugs, except in the areas of new features, which tend to be buggy at first, until the bugs are found and the unit tests catch up. The recent jump represents the integration of the new expression lexer, parser and evaluator, which is exhaustively tested.
The add rate and resolve rate usually track each other well, except in the lead up to a release, where the testing is greater. There is always a shift toward fixing right before a release.
