Taskwarrior Statistics

Interesting statistics from the taskwarrior web site and repository. These data form a baseline against which to measure changes. This page will be updated after the end of each month.

Visits to the various web sites.
Note that only downloads directly from taskwarrior.org are now recorded. SourceForge (twsf) is no longer a contributor. Note that Taskserver and the PDFs now account for half the download traffic.
Total downloads by version. This chart is indirectly measuring the time between releases, and shows that 1.9.4 and 2.3.0 were the primary version for over a year. More recently multiple versions are simultaneously available. HomeBrew, for example, downloads the tarballs from us, and the version obtained is dependent on the freshness of the local brew repository.
Total downloads by minor version. As more and more people download binary packages, which we do not provide, the downloads are gradually decreasing as those packages become more available and up to date.
Total downloads by major version.
Development began in 2006, but used CVS, then Subversion. That history was not brought into git, but consisted of 790 commits. The largest spike represents Timewarrior development.
This chart shows how the test suite sizeѕ change over time for all projects, and should generally show a migration towards libshared. This means overall tests should decrease while libshared is integrated, then will rise more slowly. We are almost at the point where libshared has the largest test suite, and will be the case as of March 2017.
The add rate and resolve rate usually track each other well, except in the lead up to a release, where the testing is greater. There is always a shift toward fixing right before a release.
This is the distribution of operating systems for our site visitors.
This char shows the downloads by version, and indicates the rate of adoption. Downloading old versions is caused by out-of-date links around the web, and when a release occurs mid-month. Due to source-package links (Slack, HomeBrew ...) older versions are still being downloaded, whereas in the past there were more distinct migrations to newer versions. This is a problem because we see more users with support questions for old versions.
Social membership numbers are still rising. G+ is dead, mostly because we never did anything there.
Downloads of feed.rss. This correlates well with the news items released.
Total top-10 404s. A recent effort to reduce 404s is being tracked here. These errors are now almost entirely speculative URL retrieval, hacking. Note that a script glitch is preventing reporting of this data.
