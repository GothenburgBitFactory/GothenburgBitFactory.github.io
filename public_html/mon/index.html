Development Task Server

This monitoring is for the development instance of the Task Server used for testing purposes.

Bytes

Average request bytes is an average size of all incoming messages from clients. These messages are usually small for a sync with no data, but can be arbitrarily large.

Idle

Idle is the percentage of time the Task Server stands doing nothing.

Service

Service time represents the average time to process a request from a client, in seconds. High load can push this time upwards.

Total Bytes

Total bytes in and out in the form of requests and responses. This is a cumulative quantity that is reset on server restart.

TPS

Transactions per second.

Transactions

Transaction count is a cumulative quantity that is reset on server restart.

Uptime

Uptime is a count of the seconds of uptime that is reset on server restart. This is a measure of server stability.

Organizations

The number of organizations supported by the dev server. This is not expected to change much.

Users

The number of user accounts on the dev server.

User Data

The amount of user data on the dev server.
